# 贡献指南

感谢你对 Gal-Tool 项目的关注！我们欢迎所有 Galgame 爱好者、开发者、汉化组成员参与贡献。

## 如何贡献

### 1. 报告问题
- 在 [Issues](https://github.com/argb6/gal-tool/issues) 中报告发现的错误
- 提出新功能建议或改进意见
- 提供工具或资源的推荐

### 2. 提交 Pull Request
1. Fork 本仓库
2. 创建功能分支 (`git checkout -b feature/amazing-feature`)
3. 提交更改 (`git commit -m 'Add some amazing feature'`)
4. 推送到分支 (`git push origin feature/amazing-feature`)
5. 打开 Pull Request

### 3. 贡献内容类型

#### 工具推荐
- **要求**：提供工具名称、版本、下载链接、简要说明、使用场景
- **格式**：Markdown 表格或列表形式

#### 网站资源
- **要求**：网站名称、URL、资源类型、访问方式（免费/付费）、语言
- **格式**：分类整理，提供简要描述

#### 文档完善
- **要求**：修正错误、补充内容、优化结构、翻译文档
- **格式**：Markdown 格式，保持风格一致

#### 代码贡献
- **要求**：Python/JavaScript/Shell 脚本，提供清晰注释
- **格式**：遵循项目代码规范

## 贡献规范

### 文档规范
- 使用 Markdown 格式
- 中文文档使用简体中文
- 保持目录结构清晰
- 图片资源放在 `assets/` 目录下

### 代码规范
- Python：遵循 PEP 8
- JavaScript：使用 ES6+ 语法
- Shell：兼容主流 Linux/macOS 环境
- 添加必要的注释和文档

### 提交信息规范
- 使用英文或中文提交信息
- 格式：`类型: 简要描述`
- 类型：`feat`、`fix`、`docs`、`style`、`refactor`、`test`、`chore`

## 资源审核标准

### 工具审核
- 工具必须与 Galgame 开发、汉化、资源管理相关
- 优先推荐开源、免费工具
- 商业工具需注明许可信息
- 确保工具安全、无恶意代码

### 网站审核
- 网站内容健康、合法
- 优先推荐稳定、长期维护的网站
- 注明访问限制（如需要注册、付费）
- 尊重版权，不推荐盗版资源

### 文档审核
- 内容准确、实用
- 格式规范、易读
- 不包含敏感、违法信息
- 尊重知识产权

## 社区准则

1. **尊重他人**：保持友好、专业的交流氛围
2. **遵守版权**：不推荐盗版工具或资源
3. **注重质量**：确保贡献内容有价值、准确
4. **持续维护**：鼓励长期维护和更新贡献内容

## 联系方式

- **Issues**：用于问题报告和功能讨论
- **Discussions**：用于一般性交流和社区讨论
- **Pull Requests**：用于代码和文档贡献

感谢你的贡献！让我们一起建设更好的 Galgame 工具生态。 🎮
